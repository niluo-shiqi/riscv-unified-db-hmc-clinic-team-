= Trusim

Trusim is an instruction set simulator generated by the cpp_hart_gen UDB backend

== Build Trusim (iss)
====
[source,bash]
----
./do build:iss CONFIG=<configuraion> [OPTIONS...]
----
----
CONFIG:
  Any fully configured UDB configuration.
  Most common:
    rv32
    rv64

OPTIONS:
  BUILDTYPE=[DEBUG, RELEASE]
    DEBUG: Includes debug symbols for debugging with GDB, no optimizations
    RELEASE: No debug symbols, optimized build output

  IGNOREUNDEFINED=[YES, NO]
    YES: Do not allow operations on bitfields with undefined bits.
    NO: Allow operations on bitfields with undefined bits.

Build output directory is ./gen/cpp_hart_gen/<CONFIG>_<BUILDTYPE>/build/
----

Example:
[source,bash]
----
./do build:iss BUILD_TYPE=DEBUG CONFIG=rv64 IGNOREUNDEFINED=YES
----
====

== Run Trusim
====
[source,bash]
----
UDB ISS
Usage: ./gen/cpp_hart_gen/rv64_Debug/build/iss [OPTIONS] [elf_file]

Positionals:
  elf_file TEXT               File to run

Options:
  -h,--help                   Print this help message and exit
  -m,--model TEXT             Hart model
  -c,--cfg TEXT               Hart configuration file
  --mm,--memory-map TEXT      Memory map file
  -p,--gdbport UINT           GDB port
  -t,--trace TEXT ...         Tracers to enable
  -l,--list-configs           List available configurations
  -g,--gdb                    GDB Debugger mode
  --halt                      Halt before execution and wait for debugger to attach
----
----
Details:
  Hart model:
    rv32, rv64

  Hart configuration file:
    Any fully configured UDB configuration, See cfg directory

  Memory Map file:
    optional memory map description file in JSON format.
    See cfg/memmap.json for an example

  GDB Debugger mode:
    Enable Trusim as a gdb server for debugging RISC-V application with gdb

  GDB port:
    optional, port gdb server listen on. Default 2159

  Tracers to enable:
    inst - instruction trace
    mem  - memory access trace
----
====
== Trusim Tracers
There are currently two tracers that can be enabled for instructions/register state during execution and memory access trace for tracing read/write accesses to SoC memory. Trace is output to the console. Support for outer trace output devices are under development.

=== Instruction trace:
====
--trace inst
[source,bash]
----
PC 8000063c addi x7, x0, 15
R x0 0
R= x7 f
PC 80000640 bne x2, x7, 68
R x2 f
R x7 f
PC 80000644 addi x3, x0, 36
R x0 0
R= x3 24
PC 80000648 addi x1, x0, 32
R x0 0
R= x1 20
PC 8000064c add x2, x1, x0
R x1 20
R x0 0
R= x2 20
----
====
=== Memory trace:
====
--trace mem
[source,bash]
----
MEM:RD 80000564
MEM:RD 80000566
MEM:RD 80000568
MEM:RD 8000056a
MEM:RD 80000550
MEM:RD 80000552
MEM:RD 80000554
MEM:RD 80000556
MEM:RD 80000558
MEM:RD 8000055a
MEM:RD 8000055c
MEM:RD 8000055e
MEM:RD 80000560
----
====

More than one tracer can be enabled and  the output will be interleved/merged in output console
